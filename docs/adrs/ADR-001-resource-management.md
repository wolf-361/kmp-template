# ADR-001: Resource Management — Libres (Strings) + Compose Resources (Images/Fonts)

- **Status:** Accepted
- **Date:** 2026-04-30

---

## Context

The project needs a multiplatform resource system with two hard requirements that drive every other decision:

1. **`S.key_name` returns a plain `String`** — usable anywhere: ViewModel, UseCase, `commonTest`, iOS Xcode Previews, Android Compose Previews. No `.get()` call, no coroutine, no setup.
2. **iOS Xcode Previews work without initialization** — string access must be zero-setup. If it needs Koin or a coroutine dispatcher to resolve strings, Xcode Previews break.

The tension is: **Compose Resources** (JetBrains, actively maintained, rich resource support) uses a `suspend getString()` API — it cannot satisfy requirement 1 or 2 without a bridging layer that degrades developer experience to the point of negating the library's advantages. **Libres** satisfies both requirements natively.

---

## Decision Drivers

1. `S.key_name` is a plain Kotlin property that returns `String` — zero boilerplate at call sites.
2. Works without Koin, without coroutines, without a test harness — in ViewModel, UseCase, test, or Xcode Preview.
3. iOS Xcode Previews receive `String` values, not resource handles.
4. Image resources need richer tooling than Libres provides (multi-density, SVG, vectors).
5. Actively maintained tooling for images and fonts.

---

## Considered Options

### Option A: Compose Resources only (rejected)

`getString(Res.string.key)` is a suspend function. Outside a Composable or coroutine context it requires a coroutine dispatcher. On iOS, SKIE exposes this as an `async` Swift function — Xcode Previews cannot call `async` functions synchronously. A `UiText` bridging class or `StringProvider` injection pattern is needed, adding complexity at every call site.

**Rejected because:** Does not satisfy requirements 1 or 2.

---

### Option B: Libres for strings + Compose Resources for images/fonts — **CHOSEN**

Libres generates a `Res.string` object at compile time where every property is a plain `String`. Compose Resources handles images, fonts, and binary files where its richer tooling justifies its async API.

## Decision Outcome

### Strings: Libres

Libres reads XML string files from `shared/src/commonMain/libres/strings/` and generates:

```kotlin
// Generated by Libres — DO NOT EDIT
object Res {
    object string {
        val err_invalid_credentials: String get() = // locale-resolved literal
        val home_title: String get() = // locale-resolved literal
    }
}
```

### The `S` alias

A single file exposes the global aliases:

```kotlin
// core/presentation/localization/Resources.kt
val S = Res.string
val I = Res.image   // Libres images (PNG/JPEG only — use Compose Resources for SVG/vectors)
```

Usage anywhere in shared code:

```kotlin
S.err_invalid_credentials       // → "Invalid credentials"  (or current locale)
S.err_network_timeout           // → "Connection timed out"
```

### String file format

```
shared/src/commonMain/libres/strings/
├── core_strings_en.xml
├── core_strings_fr.xml
├── auth_strings_en.xml
├── auth_strings_fr.xml
└── {feature}_strings_{locale}.xml
```

```xml
<!-- auth_strings_en.xml -->
<resources>
    <string name="err_invalid_credentials">Invalid credentials</string>
    <string name="auth_email_placeholder">Email address</string>
    <string name="auth_submit">Sign in</string>
</resources>
```

### Images/Fonts: Compose Resources

Compose Resources handles images, fonts, and arbitrary files where multi-density, SVG, and vector support matter:

```
shared/src/commonMain/composeResources/
├── drawable/
│   ├── ic_logo.svg
│   └── ic_placeholder.png
└── font/
    └── inter_regular.ttf
```

Used in the Compose UI layer only:

```kotlin
// composeApp — Compose Composable context
Image(painter = painterResource(Res.drawable.ic_logo))
val font = FontFamily(Font(Res.font.inter_regular))
```

Images and fonts are never accessed from shared ViewModels or domain code.

### State shape

```kotlin
data class AuthState(
    val emailError: String? = null,   // ← plain String, resolved in ViewModel
    val globalError: String? = null,
) : UiState
```

```kotlin
// ViewModel
failure<AuthError.InvalidCredentials> {
    updateState { it.copy(emailError = S.err_invalid_credentials) }
}
```

---

### iOS Xcode Previews

Because `S.key_name` is a plain Kotlin property, SKIE exposes it as a Swift computed property. It works in Xcode Previews without any initialization:

```swift
// UI-only preview — pass literal or S property
#Preview {
    LoginView(state: AuthState(emailError: S.shared.err_invalid_credentials))
}

// Or just a literal — state already holds String
#Preview {
    LoginView(state: AuthState(emailError: "Invalid credentials"))
}
```

For ViewModel previews (where the ViewModel must be created):

```swift
#Preview {
    KoinPreviewHelper.start()   // minimal Koin with mocks — needed for ViewModel DI
    return LoginContainerView()
}
```

`KoinPreviewHelper` is defined in `iosApp/` and starts Koin with lightweight mock repositories and a `PrintLogger`.

---

### Android Compose Previews

`S.key_name` works in Compose Previews without setup — Libres strings are compile-time constants:

```kotlin
@Preview
@Composable
fun LoginScreenPreview() {
    LoginScreen(state = AuthState(emailError = S.err_invalid_credentials))
}
```

### Error mapping in UiMappers

```kotlin
// features/auth/presentation/mappers/AuthUiMappers.kt
fun AuthError.asString(): String = when (this) {
    is AuthError.InvalidCredentials -> S.err_invalid_credentials
    is AuthError.EmailAlreadyExists -> S.err_email_already_exists
    else -> S.err_unknown
}
```

---

## Rejected Alternatives

| Alternative | Reason rejected |
|:---|:---|
| Compose Resources for strings | `getString()` is suspend; breaks iOS Xcode Previews; requires bridging |
| Compose Resources + UiText bridge | State holds un-resolved references; every UI consumer needs `.resolve()` |
| Compose Resources + StringProvider injection | `S.get(Res.string.key)` syntax — not `S.key`; Xcode Previews need Koin |
| Libres for images too | Supports PNG/JPEG only; no SVG, no multi-density vectors |
| Raw `expect/actual` strings | Massive boilerplate; no locale tooling |
